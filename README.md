# kerrproblems.com

A living database of open problems in **mathematical relativity**, focused on **Kerr and Kerr-related** rotating black-hole models (exact Kerr, near-Kerr vacuum, Kerr–Newman, Kerr–de Sitter, Kerr–AdS, NHEK, and related families). The site keeps a **physics-facing** lane distinct from **pure-math** motivation via orthogonal tags.

**→ [kerrproblems.com](https://kerrproblems.com)**

## Taxonomy and formal verification upgrade

- **Five clusters** (Exterior stability, Interior / SCC, Extremal, Rigidity / Uniqueness, Spectral / scattering) remain the primary navigation.
- **Orthogonal YAML tags** on every problem: `family`, `asymptotics`, `coupling`, `equation_level`, `regime`, `relevance`, `fv_suitability`, `fv_reason`, `progress_summary`, `dependencies`, optional `caution_note` / `related_families_note`.
- **`/formal-verification`** explains the FV rubric and highlights problems **K-501–K-510** plus other high/medium suitability entries.
- **Machine checks**: `npm run validate` (see `scripts/validate_problems.mjs`). Schema reference: `data/schema.md` and `data/schema.json`.
- **JSON export**: `public/problems-db.json` is generated on `npm run build` for tooling.

## Scripts

| Command | Purpose |
|--------|---------|
| `npm run validate` | Validate all `problems/*.yaml` |
| `npm run migrate:taxonomy` | Legacy → canonical YAML (already applied in-repo) |
| `npm run migrate:formal-verification` | Backfill missing `fv_reason` defaults |
| `npm run export:problems` | Write `public/problems-db.json` only |
| `npm run check` | `validate` then full `build` |

## Structure

```
problems/        one YAML file per problem
data/            clusters.yaml, schema.md, schema.json
src/             Astro site source
public/          static assets (+ generated problems-db.json)
scripts/         validate, migrate, export
```
