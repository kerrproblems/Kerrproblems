# Changelog — KerrProblems expansion (April 2026)

## Summary

- Added **100** manifest-sourced problem stubs as **K-601–K-700** under `data/problems_provisional/`, all with `publish: false` and **empty references**, because the bundled table did not include citation packets.
- Introduced **`research_state`** (`solved_in_literature` | `open_in_literature` | `high_value_unformalized_direction`), optional **`publish`**, passthrough **`solution_pointer`** / **`origin_type`**, and repo-relative **`yamlRelativePath`** in the normalized runtime object (GitHub edit links).
- **Loader** merges `problems/` and `data/problems_provisional/`; **`getPublishedProblems()`** powers the main index, clusters, RSS, and home highlights; **`getProvisionalProblems()`** powers `/audit/provisional-problems/`.
- **Editorial validator** scans both trees; reference-count rules are relaxed for provisional / `publish: false`. **`solved`** requires **`solution_pointer`** with theorem text and attribution/citation key.
- **Ingestion**: `data/expansion_from_manifest.tsv` + `node scripts/ingest_expansion_manifest.mjs` (npm `ingest:expansion`). Audit sidecar: `docs/audit/expansion_ingest_notes.md`.

## Editorial constraints honored

- No fabricated URLs or theorem claims; manifest “solved” hints map to **`research_state: solved_in_literature`** with **`theorem_status: needs_review`** (not `solved`) until `solution_pointer` and primaries exist.
- Numeric footnotes from the source table are **not** reproduced as citations.

## Follow-up for maintainers

- Curate **≥2 references** (≥1 primary) per entry, then set `publish: true` and move YAML to `problems/` when ready.
- Re-run `npm run validate` after edits; `docs/audit/problem_validation_report.md` is regenerated by `validate_editorial.mjs`.
